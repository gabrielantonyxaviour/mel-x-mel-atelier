# Mel x Mel Atelier

Hackathon: Melius Challenge

Idea: **Three Disagreeing Mels Build One Campaign**

This repo is the public execution wrapper for a Melius-native submission. The canonical creative artifact is intended to be a Melius canvas where three Mel personas generate conflicting campaign directions, an editor Mel critiques and merges the winner, and Gabriel preserves one visible human taste edit.

## Thesis

Melius is not just a generator. It is the room where creative judgment becomes visible.

## Demo Wrapper

```bash
npm install
npm run dev
npm run build
```

The Vite/React wrapper renders the judge-facing case study: the messy brief, three Mel branches, editor decision quote, human edit, demo path, and judging criteria mapping.

## Submission Packet

- `TEAM.md` records the assigned people, Chrome profile directories, emails, and ownership roles.
- `BUILD_PLAN.md` records the scope, stack, demo path, and timeboxed milestones.
- `REPO_PLAN.md` records the public repo creation and push plan.
- `SUBMISSION_PORTAL_PLAN.md` records Contra portal evidence and remaining submission fields.
- `EXECUTION_PACKET.md` is the source of truth for README copy, demo script, video script, judging criteria mapping, links, and final checklist.
- `PROGRESS.md` is the timestamped execution log.
